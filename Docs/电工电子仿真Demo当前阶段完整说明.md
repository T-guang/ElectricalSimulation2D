## 1. 这个 Demo 当前已经做到什么程度

目前项目已经从“空白 UI”推进到一个可以实际演示的电工电子仿真原型。它的核心能力有四块：

1. 顶部导航：可以在 `模拟电路`、`图纸集`、`仿真广场`、`元器件百科`、`个人中心` 五个页面之间切换。
2. 模拟电路：左侧有家庭电路组件和工业电路组件，用户可以添加元件、移动元件、连接端点、删除元件、删除线路、启动简单仿真。
3. 图纸集：可以展示已经导入的 PNG 图纸，按工业/家庭大类和初级/中级/高级分类筛选，点击图纸可以预览，进入配置后会把图纸缩小放到模拟电路场景中作为参考图。
4. 资料和配置：项目中已经建立了组件资产、图纸配置表、脚本结构和说明文档，后续可以继续补充真实器件截图或更多图纸。

当前 Demo 的目标不是计算真实电压、电流、功率、短路保护动作曲线，而是用“端点连接 + 简单通断逻辑 + UI 状态反馈”的方式，把学生最容易理解的电路搭建流程表现出来。

## 2. 从 0 开始理解这个项目

### 2.1 第一步：打开项目

1. 打开 Unity Hub。
2. 点击 `Open`。
3. 选择项目目录：`E:\Projects\Unity\ElectricalSimulation2D`。
4. 打开场景：`Assets/Scenes/Demo.unity`。
5. 点击 Unity 顶部的播放按钮进入运行模式。

运行后默认会进入 `模拟电路` 页面，并自动生成一个家庭照明示例电路。这个默认示例包含：

- 220V 电源
- 单相电能表
- 2P 空气开关
- 单开单控开关
- 电灯泡
- 电风扇
- 已连接好的线路

这样打开 Unity 后不需要从零摆元件，就可以直接看到“能演示”的内容。

### 2.2 第二步：认识界面结构

界面从上到下、从左到右可以这样理解：

1. 顶部导航栏：负责页面切换。
2. 左侧控件池：放所有可拖拽或点击生成的元件。
3. 中间画布：放置元件、接线、运行仿真。
4. 上方工具条：开始仿真、删除选中线、删除所有线、设置导线颜色、选择线条样式、保存/导入图纸等。
5. 右侧浮动图纸面板：从图纸集进入配置后出现，可以拖动、放大、缩小、关闭。

### 2.3 第三步：添加元件

添加元件有两种方式：

1. 点击左侧控件池里的元件按钮：元件会出现在画布中心附近。
2. 从左侧控件池拖动元件到画布：元件会在释放鼠标的位置生成。

生成之后，元件本身可以继续拖动。拖动时，已经连接到它的导线会跟随刷新。

### 2.4 第四步：连接线路

连接方式已经改成更直观的“预览线”模式：

1. 点击第一个端点。
2. 鼠标移动时，会出现一条跟随鼠标的预览线。
3. 点击第二个端点。
4. 完成一条正式导线。

导线颜色不是固定红色，而是根据端点类型自动判断：

- 火线、输入端、输出端、线圈 A1：红色。
- 零线、线圈 A2：蓝色。
- PE 保护地：绿色。
- 其他通用端点：使用起点端子的颜色。

导线样式目前支持：

- 自定义直角线：默认样式，适合电工图纸展示。
- 直线：作为备用样式。

### 2.5 第五步：删除元件或线路

目前删除逻辑已经实现：

1. 单击元件，元件进入选中状态。
2. 按 `D` 或 `Delete` 删除该元件。
3. 删除元件时，它相关的导线也会一起删除。
4. 单击一条线路，线路会高亮。
5. 按 `D` 或 `Delete` 删除选中的线路。

这解决了之前“新增电器不能删除”和“线路不能单独删除”的问题。

### 2.6 第六步：启动简单仿真

点击 `开始仿真` 后，程序会运行简单电路连通性判断。

当前仿真逻辑可以理解成：

1. 找到电源或电能表上的火线端点。
2. 找到电源或电能表上的零线端点。
3. 根据导线和元件内部通断关系建立一张连接图。
4. 从火线端点向外搜索，找到所有带火线电位的端点。
5. 从零线端点向外搜索，找到所有带零线电位的端点。
6. 如果火线集合和零线集合直接重叠，则判断为短路。
7. 如果某个负载同时接到了火线和零线，则认为它已经动作。

可以被点亮或动作的负载包括：

- 电灯泡
- 电风扇
- 电动机
- 指示灯
- 接触器线圈类元件

开关、熔断器、空气开关、端子排等本身不是负载，它们的作用是改变线路是否连通。

## 5. 每一个脚本的功能说明

下面按“基础数据 -> 元件显示 -> 连线 -> 仿真 -> UI -> 编辑器生成”的顺序介绍。

### 5.1 `ElectricalEnums.cs`

路径：

`Assets/Scripts/Core/ElectricalEnums.cs`

作用：定义项目中最基础的枚举和端点数据结构。

里面主要有：

`ComponentKind`

表示元件类型。例如：

- `PowerSource`：电源
- `Switch`：普通开关
- `TwoWaySwitch`：双控开关
- `PushButton`：按钮
- `Fuse`：熔断器
- `Breaker`：空气开关
- `EnergyMeter`：电能表
- `Lamp`：电灯
- `Fan`：风扇
- `Motor`：电机
- `ContactorCoil`：接触器/继电器线圈
- `Indicator`：指示灯
- `TerminalBlock`：端子排

`ComponentCategory`

表示元件属于哪一类：

- `Household`：家庭电路组件
- `Industrial`：工业电路组件

`TerminalRole`

表示端点的电气角色：

- `Generic`：通用端点
- `Phase`：火线/相线
- `Neutral`：零线
- `ProtectiveEarth`：保护地
- `Input`：输入端
- `Output`：输出端
- `CoilA1`：线圈 A1
- `CoilA2`：线圈 A2

`WireStyle`

表示导线样式：

- `Straight`：直线
- `Orthogonal`：直角线

`TerminalDefinition`

表示一个端点的配置，包括：

- `id`：程序内部用的端点 ID。
- `label`：界面上显示的端点名字。
- `role`：端点角色。
- `normalizedPosition`：端点在元件图片上的归一化坐标。
- `color`：端点显示颜色。

这里最关键的是 `normalizedPosition`。它不是像素坐标，而是 0 到 1 的比例坐标。例如：

- `(0, 0)` 表示元件左下角。
- `(0.5, 0.5)` 表示元件中心。
- `(1, 1)` 表示元件右上角。

这就是后续真实器件截图接线的核心：图片只是背景，端点坐标由 `TerminalDefinition.normalizedPosition` 决定。

### 5.2 `ComponentDefinition.cs`

路径：

`Assets/Scripts/Core/ComponentDefinition.cs`

作用：定义一个“可拖出来的元件模板”。

每个组件资产都是这个类型，例如：

- `Lamp_220V.asset`
- `Breaker_2P.asset`
- `AC_ThreePhase_Power.asset`
- `Contactor_KM_380V.asset`

它包含两类信息。

第一类是控件池和外观信息：

- `displayName`：显示名称。
- `category`：家庭组件还是工业组件。
- `kind`：元件类型。
- `bodyColor`：没有图片时的底色。
- `accentColor`：动作后的强调色。
- `sprite`：真实截图或绘制图片。
- `size`：元件在画布上的显示尺寸。

第二类是电气信息：

- `startsClosed`：默认是否闭合。
- `togglable`：是否可以双击切换通断。
- `controlledByTag`：预留字段，后续可以做接触器联动。
- `outputTag`：预留字段，后续可以做输出联动。
- `terminals`：端点列表。

简单理解：一个元件能长什么样、叫什么、有哪些端点、默认通不通，全部由 `ComponentDefinition` 决定。

### 5.3 `CircuitComponent.cs`

路径：

`Assets/Scripts/Core/CircuitComponent.cs`

作用：画布上真正出现的“元件实例”。

`ComponentDefinition` 是模板，`CircuitComponent` 是被放到画布上的实体。

它负责：

1. 根据模板初始化元件。
2. 生成端点按钮。
3. 根据 `sprite` 显示真实器件图片。
4. 支持拖动元件。
5. 支持点击选中元件。
6. 支持双击切换开关状态。
7. 根据是否动作刷新显示状态。

关键流程：

1. `Initialize()` 接收一个 `ComponentDefinition`。
2. 读取 `definition.size` 设置元件大小。
3. 如果 `definition.sprite` 不为空，就把图片显示出来。
4. 调用 `BuildTerminals()` 根据 `definition.terminals` 生成端点。
5. 端点会挂载 `TerminalView` 脚本。

真实模型导入时，就是这个脚本把图片和端点组合在一起。

### 5.4 `TerminalView.cs`

路径：

`Assets/Scripts/Core/TerminalView.cs`

作用：负责单个接线端点的点击和显示。

每个端点都是一个小的 UI 图形，同时也是一个可点击对象。

它保存：

- 所属元件 `Owner`
- 端点 ID `TerminalId`
- 端点标签 `Label`
- 端点角色 `Role`
- 端点颜色 `TerminalColor`

用户点击端点时，`TerminalView.OnPointerClick()` 会把这个端点交给 `WorkspaceController.HandleTerminalClicked()`。也就是说，端点本身不创建导线，它只是把“我被点了”告诉画布控制器。

端点连接判断并不是靠图片像素，也不是靠物理触发器碰撞，而是靠这个 `TerminalView` 对象本身。

### 5.5 `WorkspaceController.cs`

路径：

`Assets/Scripts/Core/WorkspaceController.cs`

作用：整个模拟电路画布的总控制器。

它负责几乎所有画布级操作：

- 生成元件。
- 管理画布上的元件列表。
- 接收端点点击。
- 生成连线预览。
- 创建正式导线。
- 选择元件。
- 选择线路。
- 删除选中对象。
- 清空画布。
- 启动仿真。
- 刷新状态栏文字。
- 根据网格吸附元件位置。

目前连接线路的流程就在这里：

1. 第一次点击端点时，保存为 `pendingTerminal`。
2. 鼠标移动时，`UpdatePreviewLine()` 绘制预览线。
3. 第二次点击端点时，调用 `wireManager.CreateWire()` 创建正式导线。
4. 创建完成后清除预览线。
5. 自动运行一次仿真。

导线颜色判断也在这里：

- `ResolveWireColor()` 根据两个端点的 `TerminalRole` 判断颜色。

这就是为什么现在蓝端点接出来会是蓝线，红端点接出来会是红线，PE 端接出来会是绿线。

### 5.6 `WireManager.cs`

路径：

`Assets/Scripts/Core/WireManager.cs`

作用：管理画布上的全部导线。

它负责：

- 创建导线。
- 删除单条导线。
- 删除某个元件相关的所有导线。
- 清空导线。
- 刷新全部导线位置。
- 元件移动后刷新相关导线。
- 给多条导线分配不同偏移量，减少重叠。

之前电灯泡的红蓝线重叠，就是因为多条线走同一路径。现在 `WireManager.ReflowOffsets()` 会给每条线分配不同的 `routeOffset`，让线路错开显示。

### 5.7 `WireView.cs`

路径：

`Assets/Scripts/Core/WireView.cs`

作用：单条导线的显示和点击。

每条导线并不是一条真正的 Unity LineRenderer，而是由多个 UI Image 矩形段拼起来的。

如果是直角线，会使用三段：

1. 从起点水平或垂直出去。
2. 中间转折段。
3. 接到终点。

## 6. 电路连接原理详细说明

### 6.1 模型图片和端点是两回事

这一点非常重要。

不管是现在的简化色块模型，还是未来的真实器件截图模型，图片本身都不参与接线判断。

真正参与接线的是端点对象 `TerminalView`。

也就是说：

- 图片负责看起来像真实元件。
- 端点负责被鼠标点击。
- 端点的 `TerminalRole` 决定导线颜色和仿真角色。
- 端点的 `normalizedPosition` 决定它显示在图片上的哪个位置。

### 6.2 真实模型导入后怎么配置端点

假设你有一个真实空开截图，导入 Unity 后想让它能接线，需要做这些事：

1. 把 PNG 放到 `Assets/Art/Components`。
2. 在 Unity Inspector 中把 Texture Type 设置为 `Sprite (2D and UI)`。
3. 找到或新建一个 `ComponentDefinition` 资产。
4. 把这个 PNG 赋值给 `ComponentDefinition.sprite`。
5. 设置 `ComponentDefinition.size`，控制它在画布里的显示大小。
6. 在 `terminals` 列表里添加端点。
7. 给每个端点设置：
   - `id`
   - `label`
   - `role`
   - `normalizedPosition`
   - `color`

举例，如果一个电源图片有五个端点 L1、L2、L3、N、PE，端点在图片上横向均匀分布，可以这样理解：

- L1：`(0.10, 0.35)`
- L2：`(0.30, 0.35)`
- L3：`(0.50, 0.35)`
- N：`(0.70, 0.35)`
- PE：`(0.90, 0.35)`

这里的数字不是随便写的，而是端点在图片宽高里的比例。

### 6.3 为什么不用触发器判断接线

当前项目是 Unity UI 2D 方案，不是物理场景里的 2D Collider 方案。

如果用触发器，会出现几个问题：

- UI 缩放后坐标处理更复杂。
- 端点容易和图片大小、Canvas 缩放混在一起。
- 只靠碰撞很难知道这个端点是 L、N、PE 还是 A1、A2。
- 保存和加载时仍然需要端点 ID。

所以当前方案更稳定：

- 每个端点是一个 UI Button。
- 点击端点时拿到明确的 `TerminalView`。
- 连接保存的是端点 ID，不是屏幕坐标。
- 元件移动后，导线根据端点的世界坐标刷新。

### 6.4 导线怎么跟随元件移动

每条导线保存两个对象：

- `StartTerminal`
- `EndTerminal`

元件被拖动时，端点作为元件的子对象会一起移动。拖动过程中 `WorkspaceController` 会调用：

`workspace.RefreshWiresFor(this)`

然后 `WireManager` 找到所有使用该元件的导线，让对应 `WireView` 重新计算线段位置。

所以导线不是固定画死的，而是动态根据端点位置刷新。

### 6.5 导线为什么可以错开

导线重叠问题通过 `WireManager.ReflowOffsets()` 解决。

大致逻辑：

1. 遍历全部导线。
2. 给每条线分配一个偏移量。
3. `WireView` 绘制直角线时使用偏移量改变中间折线位置。
4. 多条线即使起点终点接近，也会稍微错开。

这就是之前电灯泡红蓝线重叠问题的处理方式。

### 6.6 仿真为什么可以判断灯泡动作

以默认家庭照明电路为例：

1. 220V 电源提供 L 和 N。
2. L 通过电能表进入空气开关。
3. 空气开关闭合后，L 到达单开开关。
4. 单开开关闭合后，L 到达灯泡 L 端。
5. 灯泡 N 端通过蓝线回到空气开关 N，再回到电源 N。
6. 仿真搜索发现灯泡同时连接到火线集合和零线集合。
7. 灯泡显示 `RUN` 或动作状态。

如果断开开关，灯泡 L 端不会被火线搜索到，所以灯泡不动作。

如果把火线直接接到零线，火线集合和零线集合会重叠，仿真返回短路提示。

### 7.1 图纸图片存放

### 7.2 图纸配置表

配置表位置：

`Assets/Blueprints/BlueprintCatalog.csv`

表头是：

`FileName,DisplayName,Category,Difficulty,RecommendedComponents`

字段含义：

这样做的好处是：

- 图片可以统一放在一个文件夹。
- 同一张图纸改分类只需要改 CSV。
- UI 里的数量可以自动统计。
- 后续可以增加更多字段，例如教学视频、说明文字、知识点标签。

1. 用户进入 `图纸集` 页面。
2. 点击某张图纸卡片。

4. 用户点击 `进入电路配置`。
5. `BlueprintController.EnterConfiguration()` 关闭弹窗。
6. 顶部导航切换到 `模拟电路`。
7. 右侧显示图纸参考面板。
8. 面板内显示该图纸图片、标题、推荐元件。

当前不做自动布线，也不做图纸识别。图纸只是参考图，学生仍然需要手动拖元件和接线。

## 9. 后续如果要替换真实器件模型，应该怎么做

这是后续最重要的扩展方向。

### 9.1 推荐流程

1. 准备真实器件截图，最好是透明背景 PNG。
2. 图片不要带太大的空白边缘，否则端点坐标不好调。
3. 放入 `Assets/Art/Components`。
4. 在 Unity 里选中图片。
5. 设置 Texture Type 为 `Sprite (2D and UI)`。
6. 找到对应 `.asset` 元件模板。
7. 把图片拖到 `sprite` 字段。
8. 调整 `size`。
9. 调整 `terminals` 中每个端点的 `normalizedPosition`。
10. 进入运行模式测试接线。

### 9.2 调端点的思路

端点坐标的判断方法：

- 水平方向用 x，范围 0 到 1。
- 垂直方向用 y，范围 0 到 1。
- 图片左边是 x=0，右边是 x=1。
- 图片底部是 y=0，顶部是 y=1。

如果端点看起来偏左，就增大 x。  
如果端点看起来偏右，就减小 x。  
如果端点看起来偏上，就减小 y。  
如果端点看起来偏下，就增大 y。  

注意：Unity UI 的锚点和 RectTransform 坐标可能让视觉上稍微反直觉，所以建议每次只调整 0.02 到 0.05。

### 9.3 不建议一开始就做复杂物理模型

当前 Demo 是 UI 仿真，不需要 3D 模型，也不需要物理碰撞。

真实器件图片 + 手动配置端点，最适合当前演示目标：

- 开发速度快。
- 看起来像真实软件。
- 接线逻辑稳定。
- 后续批量增加组件比较容易。
