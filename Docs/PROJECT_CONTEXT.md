# 项目背景 (Project Context)

## 项目名称
Unity 电工数字学生仿真系统

## 当前开发阶段
目前阶段优先开发 **UI（用户界面）** 和 **图纸管理功能**。包括但不限于图纸的保存弹窗、导入图纸列表、删除图纸功能，以及完善图纸名称、保存时间、文件路径等元数据管理。

## 已经实现的功能
1. 画布交互：基础的元件生成、拖拽移动、双击切换状态。
2. 画布控制：鼠标滚轮缩放，鼠标中键/空格平移。
3. 接线系统：端子间的连线，支持正交和直线路由，导线颜色切换。
4. 状态与撤销：提供基础的撤销(Undo)与重做(Redo)功能。
5. 数据持久化：支持将当前画布状态序列化为 JSON 格式的图纸文件，并存入本地；支持加载和删除历史文件。
6. UI 页面框架：具备模拟电路、内置图纸集、元器件库等多标签页的路由切换系统。
7. 基础仪表测量：部分测试仪器（万用表、示波器）的底层显示框架。

## 暂时不要做的功能
1. 不要重构整个接线系统。
2. 不要做完整、复杂的电路物理方程式仿真。
3. 不要做标准图纸模板自动生成真实电路。
4. **绝对不要大改项目架构**，除非有极其必要的理由且经过明确确认。

## 当前重点模块
重点集中在 UI 层，尤其是 `DemoUIController` 及其调用的各种弹窗与面板组件（如 `SaveBlueprintDialog`、`ImportBlueprintPanel` 等），以及负责存储逻辑的 `SaveLoadService`。

## 后续开发注意事项
- 本项目遵循“分阶段推进”的原则，新增功能应尽可能在原架构上扩展。
- 对于 UI 修改，应保证操作流畅和提示信息的准确性。
- 代码编写时保持模块间的松耦合，不要强行把 UI 逻辑写到核心控制器中。

## 哪些脚本不能随便大改
以下核心脚本如果发生变动，可能会导致系统崩溃或重大 Bug，修改前必须慎重：
- `WorkspaceController.cs` (上帝类，包含大量的组件管理与画布逻辑)
- `SaveLoadService.cs` (存档读写核心，修改序列化字段会破坏旧存档)
- `WireManager.cs` / `CircuitComponent.cs` (元件和接线的交互逻辑强耦合，容易引起连线错误)
